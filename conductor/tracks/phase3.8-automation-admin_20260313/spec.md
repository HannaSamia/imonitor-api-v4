# Specification: Phase 3.8 — Automation & Admin

**Track ID:** phase3.8-automation-admin_20260313
**Type:** Feature (migration)
**Created:** 2026-03-13
**Status:** Draft

## Summary

Migrate four remaining REST modules from v3 to NestJS v4: AutomatedReport (scheduled report configuration), AuditLog (network audit data queries), Utility (internal consolidation + health check), and Deployment (admin tooling for module/field setup). All endpoints must achieve full functional parity with v3.

## Context

This is the final REST-layer migration phase before Socket.IO (Phase 4) and background jobs (Phase 3.9). The v3 source for these modules lives in `../imonitor-v3-api/src/`. TypeORM entities for AutomatedReport already exist in v4. The other modules rely primarily on raw SQL via `LegacyDataDbService` / `LegacyEtlDbService`.

## User Story

As a system administrator / NOC engineer, I want to configure automated report delivery, query audit logs, run data consolidation, and manage module metadata so that reporting, compliance, and platform operations work correctly in v4.

## Acceptance Criteria

### AutomatedReport (`api/v1/automatedreports`)
- [ ] POST `/` — create automated report with email/SFTP config; SFTP password AES-encrypted at rest
- [ ] PUT `/:id` — update automated report (replace email/SFTP records)
- [ ] DELETE `/:id` — soft delete (isDeleted=1, deletedOn timestamp)
- [ ] PUT `/change/status/:id` — toggle isActive; set activatedOn timestamp
- [ ] GET `/user` — list current user's non-deleted reports (id, title, isActive)
- [ ] GET `/report/:id` — list reports linked to a given report ID
- [ ] GET `/:id` — full detail with decrypted SFTP password
- [ ] All endpoints: JWT + PrivilegeGuard; DTOs with class-validator; Swagger decorators

### AuditLog (`api/v1/auditlog`)
- [ ] GET `/:fromDate/:toDate/:operation` — return `{header, body}` tabular format; queries `iMonitorData.V3_audit_logs_stats`
- [ ] GET `/:id/:request` — decompress and return single request or response blob via `UNCOMPRESS()`
- [ ] GET `/operations` — list distinct operations from `iMonitorData.V3_audit_logs_operations`
- [ ] All queries via `LegacyDataDbService`; column headers fetched from `core_modules_tables` + `core_tables_field`

### Utility (`api/v1/utilities`)
- [ ] GET `/ping` — public health-check endpoint, returns `{ status: 'ok' }`
- [ ] POST `/consolidate` — trigger stat consolidation for given tables + date; validates via API key (`ApiKeyGuard`)
- [ ] Consolidation logic: fetch ETL flow config, group-aggregate minutely → hourly → daily using SELECT INTO OUTFILE + LOAD DATA INFILE pattern via raw SQL; update `V3_consolidation_check`

### Deployment (`api/v1/deploy`)
- [ ] POST `/tablefields/:tableType` — auto-introspect INFORMATION_SCHEMA and insert missing `core_tables_field` records; JWT protected
- [ ] POST `/module` — insert new module into `core_modules` + bulk-insert `core_privileges` for all users; JWT protected

## Out of Scope

- Actual automated report execution and scheduling (cron + worker script → Phase 3.9)

## Technical Notes

- AutomatedReport `GET /:id` must AES-decrypt SFTP password using `EncryptionHelperService.decrypt()` (mirrors `dbDecrypt()` in v3)
- AuditLog header generation fetches column metadata from `core_modules_tables` JOIN `core_tables_field` filtered by `tableName='V3_audit_logs_stats'`
- Utility `/consolidate` accesses `EtlV3_2` for flow config and `iMonitorData` for stat tables — both via raw SQL services
- Deployment `/tablefields` uses `INFORMATION_SCHEMA.COLUMNS` query via `LegacyDataDbService`
- Utility guard: `ApiKeyGuard` checks `access_token` header against `core_sys_config.utilityApiKey`
