# Specification: Phase 3.3.1 — Reports Module

**Track ID:** phase3.3.1-reports_20260304
**Type:** Feature
**Created:** 2026-03-04
**Status:** Draft

## Summary

Migrate the Reports module from Express.js v3 to NestJS v4, covering 32 endpoints: report CRUD, sharing, 9 chart generation types, and 11 export endpoints (7 full-report + 4 per-tab).

## Context

Reports is the largest module in iMonitor — it powers the core statistics and data visualization experience. Management stakeholders and NOC engineers use it to build custom reports from privileged data tables, generate charts, share reports across teams, and export in multiple formats. This is the first of four Phase 3.3 sub-tracks (3.3.1 Reports, 3.3.2 WidgetBuilder, 3.3.3 QBE, 3.3.4 Export).

## User Story

As a NOC engineer or management stakeholder, I want to create, view, share, and export reports with charts so that I can analyze telecom network statistics and share findings with my team.

## Acceptance Criteria

- [ ] All 32 v3 report endpoints migrated with identical request/response contracts
- [ ] Report CRUD: create, update, rename, delete, list, get-by-id, favorite toggle, ownership transfer
- [ ] Sharing: share report with users, get shared report, save shared report as own
- [ ] Chart generation: tabular, query, pie, doughnut, trend, vertical bar, horizontal bar, progress, exploded progress, generateChartByType
- [ ] Export: CSV, JSON, HTML, PDF, PNG, JPEG, Excel (full report + per-tab HTML/PDF/PNG/JPEG)
- [ ] Privilege checks: users only see tables they have access to, module admin checks for delete/transfer
- [ ] Close tab (delete chart from report)
- [ ] Privileged statistics side menu endpoint
- [ ] Swagger decorators on all endpoints
- [ ] DTOs with class-validator for all inputs
- [ ] Unit tests for ReportsService (mocked repos)
- [ ] Queries against iMonitorData via LegacyDataDbService (raw SQL), report metadata via TypeORM

## Technical Notes

- v3 report service is ~1,559 lines; chart generation functions are imported from separate files
- JSON-stringified columns: `tables`, `globalFilter`, `orderBy`, `control`, `operation`, `compare`, `options` — stored as TEXT, parsed at runtime
- Chart generation executes dynamic SQL against iMonitorData (not iMonitorV3_1)
- Export flow: fetch report → generate charts → render HTML template → convert format (puppeteer for PDF/PNG/JPEG)
- ExportHelperService already handles Excel; needs extension for CSV, JSON, HTML, PDF, PNG, JPEG
- QueryBuilder service (generates SQL from report config) will be implemented as part of this track
