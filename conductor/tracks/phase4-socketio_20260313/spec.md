# Specification: Phase 4 — Socket.IO Gateways

**Track ID:** phase4-socketio_20260313
**Type:** Feature
**Created:** 2026-03-13
**Status:** Draft

## Summary

Migrate all 6 v3 Socket.IO handlers to NestJS WebSocket gateways using
`@nestjs/websockets` + `@nestjs/platform-socket.io`, the Redis adapter for
cluster support, and a `WsJwtGuard` for handshake authentication.

## Context

The product is an enterprise telecom monitoring platform. Real-time dashboards,
connectivity status, ETL-triggered chart refreshes, and observability alerts all
depend on Socket.IO. Phase 3 completed all HTTP modules but left Socket.IO as
plain stubs. This phase wires up the full WebSocket layer with parity to v3.

## User Story

As a NOC engineer or dashboard stakeholder, I want real-time dashboard charts,
connectivity status, and observability alerts to refresh automatically in my
browser so that I see live data without polling.

## Technical Notes

- NestJS gateways use `@WebSocketGateway({ namespace: '/x', transports: ['websocket'] })`
- `WsJwtGuard` reads token from `socket.handshake.auth.token`, validates with
  `JwtService.verify()`, attaches decoded payload to `socket.data.user`;
  throws `WsException('Unauthorized')` on failure
- Redis state management mirrors v3: `scanStream` for cross-worker key enumeration;
  `lrange`/`rpush` for dashboard chart lists, `set`/`get` for notification/
  connectivity socket maps
- EtlGateway uses Bottleneck (`maxConcurrent: 10`, `minTime: 100 ms`) backed by
  the existing `ioredis` client (same as v3)
- `DashboardGateway.getClientsFromStorage()` also inserts a row into
  `V3_opened_dashboards_stats` (iMonitorData) via `LegacyDataDbService`
- Cross-gateway communication (EtlGateway → DashboardGateway) is handled by
  injecting `DashboardGateway` into `EtlGateway` (both in the same `GatewaysModule`)
- `NotificationService.processChartNotification()` is called from EtlGateway
  after successful chart generation (skip for COMPARE_TREND, TREND, PIE types)
- All gateways co-located under `src/gateways/` with a single `GatewaysModule`
